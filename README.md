# 是什么

作为一个产品经理，如果你的主要工作就是画原型和写PRD，那么这几个skills将帮你完成从一个需求的梳理到最终输出高保真原型的PRD的全流程

也可以搭配MCP使用，直接把生成的HTML原型输出为设计稿

# 为什么

来自本人真实跑出的产品经理AI时代工作流
也许AI现在在agent、代码层级，还负责不了大规模的业务系统，但这关你一个不报销token的小公司产品经理什么事？如果你只是每天画画原型写写PRD，相信我，这些事情交给AI足够了。用好AI，在产品经理最好的时代里，别让自己被AI取代。

> 把工作用小于1/100工资的价格外包给AI，然后自己摸鱼不香吗？

> 最初，没有人在意这场灾难，这不过是一个模型，一次进步，一个公司的裁员，一种职业的消失。直到这场灾难和每个人息息相关。

# 有什么

包含4个skills

|名称|作用|
|---|---|
|AIGPRD(C)|把清晰的需求转化为PRD，适用于你已经清楚知道这个需求该怎么做的情况。你可以跟AI描述完你的需求详情，或者直接丢一张网页截图，让AI帮你写PRD|
|AIGPRD(V)|把模糊的需求转化为PRD，如果你还不能明确需求，也告诉AI吧，他会通过对话的方式引导你明确需求，然后写好PRD|
|AIGPP|PRD写完之后，可以直接转化为高保真的HTML原型，搭配MCP和design.md，你甚至可以干掉隔壁工位的UI/UE|
|AI评审（未完成）|PRD生产完成之后的评审|
