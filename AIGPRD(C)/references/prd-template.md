# 1. 概述（为什么做）

## 1.1 需求背景

为什么要做这个需求，介绍一下过往需求遇到的问题（非必要），当前需求的产生的原因（必要）

## 1.2 需求概述

对当前需求做简略性的描述，尽量在3句话之内把需求描述清楚

## 1.3 需求目标

对需求的业务目标和用户目标做基本的说明

### 1.3.1 业务目标

当前需求对公司业务有什么帮助，对想要达成的业务目标进行描述，并给出相关指标及实现路径

### 1.3.2 用户目标

当前需求对使用产品的用户有什么帮助，并给出相关指标

## 1.4 产品框架

描述产品的整体结构，模块划分等。

## 1.5 功能列表与模块

详细的功能清单及功能归属的产品模块，与之前功能的耦合关系

## 1.6 目标用户（非必要）

对系统涉及的目标用户进行说明

## 1.7 名词说明（非必要）

针对文档中出现的业务名词，特有名词，自定义名词进行解释和说明

## 1.8 角色及权限（非必要）

对系统设计的角色和权限进行说明。

## 2.1 功能流程

对功能主流程、子流程进行说明

### 2.1.1 主流程

绘制产品主流程图

### 2.1.2 子流程（非必要）

绘制产品子流程图

### 2.1.3 数据流图（非必要）

功能中数据的流动、处理过程、数据存储以及系统与外部实体的交互。

## 2.2 需求描述

对功能的详细描述

### 2.2.1 用户故事（非必要）

描述系统参与的角色的使用功能（非必要）

### 2.2.2 前置条件（非必要）

功能的前置条件，比如需要登录或特定角色权限，以及功能的数据依赖和其他功能依赖

### 2.2.3后置条件（非必要）

功能的后置条件，比如执行操作后更新列表数据、弹出弹窗、toast提示等。

## 2.3 界面及交互

功能的界面、元素的说明及其交互

## 2.4 异常/分支流程（非必要

功能的异常流程与分支流程

# 3. 用户体验设计

## 3.1 全局说明

### 3.1.1 全局异常处理

对全局的异常的处理方式进行说明，如网络异常、后台服务异常等

* 网络异常：提示“请检查网络连接”，支持“重试”操作；
* 服务超时：提示“系统响应超时，请稍后重试”；
* 权限异常：提示“无操作权限”；
* 系统异常：提示“系统开小差啦，请联系管理员”，并记录日志；
* 数据异常：提示“数据异常或不存在”；

### 3.1.3 全局交互

对全局的数据交互和用户交互进行简单说明，如提示弹窗、提示吐司、缺省数据显示、操作弹窗等。

* 操作成功：toast提示“操作成功”，自动消失；
* 操作失败：弹窗提示错误详情；
* 加载状态：显示全局loading动画；
* 表单保存后自动返回上一页并提示；
* 删除操作需二次确认；
* 异步操作时按钮置灰防止重复提交；
* 空状态：显示插画和文字提示“暂无数据”。

## 3.2 普通列表规则

对产品牵涉到的通用表单规则进行简单说明，如排序规则、分页、跳页、缺省值、统计等。

* 默认分页20条，可调整为10/50/100；
* 支持排序、模糊搜索、多条件筛选；
* 空数据时显示“暂无数据”；
* 列表顶部展示统计信息；
* 支持Excel导出；
* 批量操作前需二次确认。
